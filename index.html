Easy To Install

Download Net install Upgrade from XenServer Web UI

Grab the latest ISO here, create your bootable USB key with:

dd if=xcp-ng-7.5.0-2.iso of=/dev/sdX bs=8M status=progress oflag=direct

Then just boot on it!

On Windows, you can use Rufus to create the bootable USB stick.

MD5: 3723d059649baf7e94824da2da2c1bbc
SHA256: f45388c8835fbb75141df004112e20861b4828872811b40ba815c506f724a798

Using the Net install ISO (only 100MiB!), create your bootable USB key:

dd if=xcp-ng-7.5.0-2-netinstall.iso of=/dev/sdX bs=8M status=progress oflag=direct

Then just boot on it!

On Windows, you can use Rufus to create the bootable USB stick.

MD5: 438f4f74380c24b4b5d13b514014e9f0
SHA256: a0b946330467d3bf5e685f67ee65fe80a9e1b4a83f51c3226c3e618d9f7115c4

Just insert the ISO and follow instructions. You'll have an upgrade panel, just select it and that's it!

After a reboot, you'll be on XCP-ng while keeping all your previous settings (Storage, VMs…)



No physical access to the XenServer host?

If you do not have access to your server or remote KVM in order to upgrade using the interactive ISO installer, you can initiate an automatic reboot and upgrade process using the following procedure:

Unpack/extract the XCP-ng ISO to a folder on a webserver. Then get the UUID of your host by running the below command:

xe host-list

Using that host UUID, as well as the URL to the folder hosting the unpacked XCP-ng ISO, run the following command to test access:

xe host-call-plugin plugin=prepare_host_upgrade.py host-uuid=750d9176-6468-4a08-8647-77a64c09093e fn=testUrl args:url=http:///xcp-ng/unpackedexample/

The returned output must be true to continue. Now tell the host to automatically boot to the ISO and upgrade itself on next reboot (using the UUID and URL from before):

xe host-call-plugin plugin=prepare_host_upgrade.py host-uuid=750d9176-6468-4a08-8647-77a64c09093e fn=main args:url=http:///xcp-ng/unpackedexample/

The output should also be true. It has created a temporary entry in the grub bootloader which will automatically load the upgrade ISO on the next boot.

It then automatically runs the XCP-ng upgrade with no user intervention required. It will also backup your existing XenServer dom0 install to the secondary backup partition, just like the normal upgrade.

To start the process, just tell the host to reboot. It is best to watch the progress by using KVM if it’s available, but if not, it should proceed fine and boot into upgraded XCP-ng in 10 to 20 minutes.

Just type this into your XCP-ng console:

bash -c "$(curl -s http://xoa.io/deploy)"

This will deploy a small VM doing all the management and backup for your XCP-ng. Follow instructions given on the screen, and you're up and running!
